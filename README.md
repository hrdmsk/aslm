ASLM UI 開発ドキュメント

概要

ASLM (Asset Library Manager) は、ローカルのUnityパッケージとBooth/Gumroadなどの外部ソースを紐付けて管理するアプリケーションです。
本UIはWindowsエクスプローラーに似た操作感を提供し、Vue 3 + Composition API + Piniaで構築されています。

ディレクトリ構成

```
src/
├── components/
│   ├── ExplorerLayout.vue  # アプリケーション全体のレイアウト枠
│   ├── AddressBar.vue      # パス表示、ナビゲーション、検索バー
│   ├── DirectoryTree.vue   # 左サイドバー：再帰的なフォルダツリー
│   ├── FileList.vue        # メインエリア：ファイル/フォルダのグリッド表示
│   └── FilePreview.vue     # 右サイドバー：選択アイテムの詳細プレビュー
├── stores/
│   └── fileSystem.js       # 状態管理：現在のパス、ファイルリスト、選択状態
└── App.vue                 # ルートコンポーネント
```

主要コンポーネントの役割

1. ExplorerLayout.vue

役割: アプリケーションの骨格を提供します。ヘッダー、サイドバー、コンテンツ、フッターの4領域を定義し、スロット (<slot>) を通じて中身を注入します。

特徴: 画面全体を使用し、スクロール領域を適切に制御します。

2. FileList.vue (Main Content)

役割: 現在のディレクトリ内のアイテムをグリッド形式で表示します。

主な機能:

グリッド表示: CSS Gridを使用し、ウィンドウ幅に応じて列数を自動調整します。

サムネイル表示:

フォルダ: 指定されたイメージ画像（現在はモック画像 motorcycle.svg）を表示。

ファイル: ファイルアイコンを表示。

連携バッジ: BoothやGumroadのソース情報がある場合、バッジを表示します。

インタラクション:

クリック: アイテムを選択し、右側のプレビューを表示。

ダブルクリック: フォルダなら階層移動。

3. FilePreview.vue (Right Pane)

役割: FileList で選択されたアイテムの詳細情報を右側に表示します。

データフロー: 親コンポーネント (FileList) から props 経由で選択されたアイテムデータを受け取ります。

表示内容:

大きなプレビュー画像（フォルダの場合）

ファイル種別、更新日時

外部ソース（Booth/Gumroad）へのリンクボタン

4. DirectoryTree.vue (Left Sidebar)

役割: フォルダ構造をツリー表示します。

特徴: 再帰コンポーネントとして実装されており、自分自身を呼び出すことで無限の階層構造を表現します。

5. AddressBar.vue

役割: 現在のパスを表示・編集したり、親階層へ移動するナビゲーション機能を提供します。

状態管理 (Pinia Store)

stores/fileSystem.js で以下の状態を一元管理しています。

currentPath: 現在表示しているディレクトリのパス。

fileList: バックエンド（Go）から取得したファイル/フォルダの配列。

changeDirectory(path): パスを変更し、ファイルリストを再取得するアクション。

UI/UX の特徴的な実装

レスポンシブな3ペイン構成:

[ツリー] - [ファイルグリッド] - [プレビュー] の3列構成。

プレビューペインはアイテム選択時のみ表示されます。

ビジュアルフィードバック:

ホバー時や選択時にWindowsライクな背景色（青系）の変化を適用。

フォルダとファイルで視覚的な表現を明確に区別（フォルダ＝画像、ファイル＝アイコン）。
